var: Function-scoped, hoisted, can be redeclared and reassigned.
let: Block-scoped, not hoisted, can be reassigned but not redeclared.
const: Block-scoped, not hoisted, cannot be reassigned or redeclared after initialization.

undefined: Typically represents the lack of assignment or the absence of a meaningful value. It is automatically assigned to variables that are not initialized.
null: Explicitly indicates the absence of a value or can be used to reset a variable or property to no value. Developers use null when they want to signify intentional emptiness.

REST API is a way for different computer systems to talk to each other over the internet. It's like a set of rules that makes it easy for developers to create and connect web services, allowing them to work together smoothly.
